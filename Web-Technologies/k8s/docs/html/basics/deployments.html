Deployments

Ein Deployment sorgt dafuer, dass immer die gewuenschte Anzahl Pods laeuft. Es bietet Self-Healing und Rolling Updates.

┌─────────────────────────────────────────────────────────────────┐
│  DEPLOYMENT: flask-webserver                                     │
│  ═══════════════════════════                                     │
│                                                                  │
│  spec:                                                           │
│    replicas: 2        ◄── "Ich will immer 2 Pods!"              │
│                                                                  │
│  ┌─────────────────────────────────────────────────────────────┐│
│  │  ReplicaSet (wird automatisch erstellt)                     ││
│  │                                                             ││
│  │   ┌─────────────┐    ┌─────────────┐                       ││

│  │                                                             ││
│  │   Wenn ein Pod stirbt ───► K8s startet automatisch neuen!  ││
│  └─────────────────────────────────────────────────────────────┘│
└─────────────────────────────────────────────────────────────────┘

Self-Healing bedeutet: Kubernetes vergleicht staendig den IST-Zustand mit dem SOLL-Zustand und korrigiert Abweichungen automatisch.

YAML deployment.yaml 
apiVersion: apps/v1
kind: Deployment
metadata:
  name: flask-webserver
spec:
  replicas: 2                       # ◄── Anzahl der Pods

  selector:
    matchLabels:
      component: webserver          # ◄── "Ich manage Pods mit diesem Label"

  template:                         # ◄── Pod-Template
    metadata:
      labels:
        component: webserver        # ◄── MUSS mit selector matchen!
    spec:
      containers:
        - name: flask
          image: meta-playlist:latest
          ports:
            - containerPort: 8001

┌─────────────────────────────────────────────────────────────────┐
│  SELECTOR + LABELS                                               │

Wichtig: Der selector.matchLabels MUSS mit den template.metadata.labels uebereinstimmen!

readinessProbe

YAML 
readinessProbe:               # ◄── "Ist der Pod bereit fuer Traffic?"
  httpGet:
    path: /
    port: 8001
  initialDelaySeconds: 5      # ◄── Warte 5s nach Start
  periodSeconds: 5            # ◄── Pruefe alle 5s

livenessProbe

YAML 
livenessProbe:                # ◄── "Lebt der Container noch?"
  httpGet:
    path: /
    port: 8001
  initialDelaySeconds: 30     # ◄── Warte 30s (App braucht Zeit)
  periodSeconds: 10
ProbeWenn fehlschlaegtGut fuer
readinessProbe Kein Traffic, Pod laeuft weiter Startup-Phase
livenessProbe K8s startet Container neu! Deadlock-Erkennung

Bash 
# Deployment erstellen
kubectl apply -f deployment.yaml

# Status pruefen
kubectl get deployments
kubectl get pods

# Skalieren
kubectl scale deployment flask-webserver --replicas=3

# Rolling Update
kubectl set image deployment/flask-webserver flask=meta-playlist:v2

# Rollback
kubectl rollout undo deployment/flask-webserver
