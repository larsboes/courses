Was ist Kubernetes?

Kubernetes (K8s) ist ein "Betriebssystem fuer die Cloud". Es verwaltet Container so wie ein OS Prozesse verwaltet.

┌────────────────────────────────────────────────────────────────────┐
│                    ANALOGIE: OS vs. Kubernetes                      │
├────────────────────────────────────────────────────────────────────┤
│                                                                     │
│   Traditionelles OS              Kubernetes                         │
│   ─────────────────              ──────────────                     │
│   Prozesse                  ───► Pods (Container)                   │
│   Dateisystem               ───► Persistent Volumes                 │
│   Netzwerk-Sockets          ───► Services                           │
│   Process Scheduler         ───► kube-scheduler                     │
│   Init System (systemd)     ───► Deployments                        │
│   Hostname/DNS              ───► Service Discovery                  │
│                                                                     │
└────────────────────────────────────────────────────────────────────┘

Diese Analogie hilft dir zu verstehen, warum Kubernetes als "Cloud OS" bezeichnet wird. Genau wie ein Betriebssystem auf deinem Computer Ressourcen verwaltet, macht Kubernetes das Gleiche fuer Container in einem Cluster.

Kernidee: Du beschreibst den gewuenschten Zustand (deklarativ), K8s sorgt dafuer, dass dieser Zustand erreicht und gehalten wird. 
┌─────────────────────────────────────────────────────────────────┐
│  IMPERATIV vs. DEKLARATIV                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  Imperativ (Docker):           Deklarativ (K8s):                │
│  ──────────────────            ─────────────────                │
│                                                                  │
│  "Starte Container A"          "Ich will 3 Instanzen            │
│  "Starte Container B"           von Container A"                │
│  "Starte Container C"                                           │
│                                                                  │
│  Du sagst WIE                  Du sagst WAS                      │
│  Du bist verantwortlich        K8s ist verantwortlich           │
│                                                                  │
│  Container stirbt?             Container stirbt?                 │
│  → Du musst neu starten        → K8s startet automatisch neu    │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘

Vorteile des deklarativen Ansatzes

  • Self-Healing: Kubernetes repariert automatisch
  • Idempotenz: Mehrfaches Anwenden der gleichen Konfiguration hat den gleichen Effekt
  • Versionierbar: YAML-Dateien koennen in Git verwaltet werden
  • Reproduzierbar: Gleiche Config = Gleiches Ergebnis
