🔍 Component Deep Dive

🖥️ Client (Frontend)

Runs in the user's browser. Responsible for:

  • Rendering the user interface (HTML/CSS)
  • Handling user interactions (clicks, form submissions)
  • Making API calls to the server using fetch()
  • Updating the DOM based on server responses

Key Files: index.html, style.css, script.js

⚙️ Server (Backend)

Runs in the Docker container. Responsible for:

  • Receiving HTTP requests from clients
  • Routing requests to the correct handler (Flask routes)
  • Processing business logic (validation, CRUD operations)
  • Returning JSON responses
  • Serving static files (HTML, CSS, JS)

Key File: app.py

Architecture Overview

The application follows a microservices-based architecture using Docker containers.

  • Frontend/Backend: A Flask web server serving the UI and providing a REST API.
  • Database: A CouchDB NoSQL database for persistent storage of playlists.
  • Communication: The Flask container communicates with CouchDB via HTTP/REST on port 5984.

Key Additions

  • Load Balancer: Distributes traffic across multiple servers
  • Multiple Containers: Horizontal scaling for high traffic
  • CDN: Caches static assets globally for faster loading
  • Managed Database: Auto-backups, replication, high availability
  • Redis Cache: Fast session storage, API response caching
