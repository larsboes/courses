Pods

Ein Pod ist die kleinste deploybare Einheit in Kubernetes - eine Gruppe von Containern, die zusammen laufen und sich Ressourcen teilen.

Container im Pod teilen sich:

RessourceBeschreibung
Network Namespace Gleiche IP, koennen ueber localhost kommunizieren
Volumes Gemeinsamer Speicher zwischen Containern
IPC Inter-Process Communication

YAML pod.yaml 
apiVersion: v1
kind: Pod
metadata:
  name: flask-pod
  labels:
    component: webserver    # ◄── Labels fuer Service-Matching
spec:
  containers:
    - name: flask
      image: meta-playlist:latest
      ports:
        - containerPort: 8001

Erklaerung:

  • apiVersion: v1 - Pods gehoeren zur Core API
  • kind: Pod - Typ der Ressource
  • labels - Key-Value Paare zur Identifikation (wichtig fuer Services!)
  • containerPort - Port auf dem die App lauscht

In der Regel: Ein Container pro Pod!

Mehrere Container nur wenn sie:

  • Zusammen skaliert werden muessen
  • Zusammen starten/stoppen muessen
  • Gemeinsame Daten brauchen
